This project involved performing a PCI DSS v4.0 scope determination and segmentation assessment for a simulated e-commerce company processing credit card transactions. I analyzed the Cardholder Data Environment (CDE) to identify systems that store, process, and transmit cardholder data,  while also mapping how CHD flows throughout the environment using a data flow diagram. During the assessment, I evaluated connected systems, security-impacting systems, shared services, and trust relationships that crossed segmentation boundaries between the Corporate Zone and the CDE. I identified how shared infrastructure such as Active Directory, WSUS patch management, logging/SIEM systems, backup infrastructure, and Jump Host administration weakened segmentation effectiveness and potentially expanded PCI DSS scope. I also conducted segmentation analysis, identified segmentation gaps, created remediation recommendations, and performed residual risk analysis. Through this project, I applied PCI DSS v4.0 concepts including scope determination, CHD flow analysis, segmentation validation, trust relationship analysis, risk assessment, tokenization, encryption, HSM functionality, administrative access control, and GRC-based security analysis.

Practical Experience:

- Cardholder Data Environment (CDE) and CHD flow analysis
- Active Directory, WSUS, SIEM, and backup infrastructure security analysis
- Trust relationship and shared services assessment
- VLAN and firewall segmentation evaluation
- Tokenization, encryption, and HSM workflow analysis

- QSA-style PCI compliance analysis and audit documentation
